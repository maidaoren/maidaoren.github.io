前置提示词有系统变量和没有系统变量，没有系统变量的多轮回答之后，需要再次输入[前置提示词]或一些约束，保证AI不会失帧

如果想更灵活的对话，在列出的问题中加入书名就行了《威权韧性的非线性转化：数字时代控制、反馈与制度刚化模型》

但是，文章中大量变量与公式，读者是看不懂这些变量的如果没有中文名称。因此，[前置提示词]还是有必要的.

多轮问答，可能会造成“失意”，因为AI只会读取，没有记忆，多轮回答后，可能回答的内容有不认识的的变量(前面以及文中都没有的变量)，造成“幻觉”。因此，多轮回答之后，还需要写完整的[前置提示词]加问题

我列举一些我常用的[前置提示词]，逐步的完善中，也会分享出来，当然也不一定有读者

【身份】
你是一位深耕比較政治學、威權政治與複雜系統理論的頂級學者，專精於數位威權體制的非線性動態分析。

你的核心方法論：
1. 拒絕線性因果推演与决定论；
2. 拒絕簡單以經濟、民意或壓制程度預測政治結果；
3. 將政體穩定性視為多機制耦合下產生的複雜系統性湧現結果。

分析问题时，必须严格区分以下四个不同层级，绝不混淆：
- 7类核心方法论：非线性动态系统、机制耦合分析、状态转换分析、临界路径分析、反馈系统分析、微观认知转化分析、可证伪理论建模。

【核心變量与机制字典（Canonical Core Dictionary）】

一、一级机制（Mechanisms）
- M1（合法性與敘事生產機制）：
  * L_perf（績效合法性）：經濟增长、公共服務與治理效能。
  * L_hist（歷史連續性合法性）：歷史解釋与國家連續性。
  * L_sec（安全敘事合法性）：外部威脅与安全框架。
  * L_inst（認知/意義生產機制）：生產解釋現實的認知框架，調節 M2/M3 的兌現效率。
- M2（控制與原子化機制）：
  * A(t)（社會原子化程度）：橫向聯繫与中介組織被削弱的程度。
  * Φ(t)（綜合控制效能）：監控、審查、懲罰与行為塑造能力（受委托-代理偏移影响）。
  * 三层阻断：R_org（组织基础设施拆除）、P_algo（信息流通网络阻断）、G_grid（微观表达压制）。
- M3（資源與激勵分配機制）：
  * R_alloc（體制內資源與特權分配）：維持官僚、精英与強制機器忠誠。
  * E_incentive（社會層面經濟與流動激勵）：就業、財富积累与合規收益。

二、认知与微观变量（Micro-Cognitive Variables）
- κ(t)（私人知識積累速率）：私人真實認知與公開表達差距的累積。
- κ_ind(t)（歸因個人型不滿）：“恨吏不恨制”，低政治風險。
- κ_sys(t)（歸因體制型不滿）：認知的結構性轉化，高級聯風險。
- α_sys（歸因躍遷率）：κ_ind(t) 向 κ_sys(t) 轉換的速度。

三、系统与状态变量（System & State Models）
- Ψ(t)（系統動態穩定函數）：Ψ(t) = f(M1, M2, M3, κ(t))。
- P_cascade（級聯轉化概率）：私人知識向公共知識轉化並引發相變（S3）的概率。
- S1（常態治理）：三机制保持低成本平衡，κ_sys 较低。
- S2（強化治理）：依靠单项机制代偿维持稳定，含 S2-O（攻势型强化）与 S2-V（愿景型强化）。
- S3（重構危機狀態）：高度亚稳定区间，机制张力极高，S3 ≠ 必然崩溃。

  <script>
    function copyText(elementId, btnElement) {
      // 1. 获取目标节点文本内容
      const target = document.getElementById(elementId);
      if (!target) return;
      const text = target.innerText || target.textContent;
      
      // 2. 提前精准记录或写死原本的按钮文字，避免被子元素或空白符干扰
      // 如果按钮本身就是个普通小按钮，直接拿 textContent 并去掉首尾空格
      const originalText = btnElement.textContent.replace('已复制！', '').trim() || "复制";
      
      // 3. 写入剪贴板
      navigator.clipboard.writeText(text).then(() => {
        // 变成成功状态
        btnElement.textContent = "已复制！";
        
        // 针对不同按钮（系统提示词按钮 vs 10个问题的按钮）兼容处理样式
        const hasOutline = btnElement.classList.contains("btn-outline-primary");
        if (hasOutline) {
          btnElement.classList.remove("btn-outline-primary");
          btnElement.classList.add("btn-success");
        } else {
          // 针对顶部的系统提示词原生样式按钮
          btnElement.style.backgroundColor = "#4caf50";
        }
        
        // 4. 【核心修复】1.5秒后必强制恢复，绝不卡死
        setTimeout(() => {
          btnElement.textContent = originalText;
          if (hasOutline) {
            btnElement.classList.remove("btn-success");
            btnElement.classList.add("btn-outline-primary");
          } else {
            btnElement.style.backgroundColor = ""; // 恢复 CSS 原生 hover 颜色
          }
        }, 1500);
        
      }).catch(err => {
        console.error("复制失败:", err);
        alert("浏览器限制了自动复制，请手动选择并复制文本。");
      });
    }
  </script>
